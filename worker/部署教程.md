# 植愈 · Cloudflare Worker 部署教程

## 为什么需要这个？

之前 AI 的密钥和 EmailJS 的 ID 都明文写在前端代码里，任何人按 F12 就能拿走。
现在改成通过 Cloudflare Worker 中转：密钥只存在 Worker 里，前端代码完全看不到。

**效果**：评委 F12 看 Network 面板，只能看到请求发到 `your-worker.workers.dev`，看不到任何密钥。

---

## 第一步：注册 Cloudflare 账号（免费）

1. 打开 https://dash.cloudflare.com/sign-up
2. 填邮箱 + 密码注册（不需要信用卡，免费额度够用）
3. 注册后登录，进入控制台

> 免费额度：每天 10 万次请求，远远超过评委演示所需。

---

## 第二步：创建 Worker

1. 登录后，左侧菜单点 **Workers & Pages**
2. 点 **Create application** → **Create Worker**
3. 名字填 `zhiyu-proxy`（随便起，会成为 URL 的一部分）
4. 点 **Deploy**
5. 部署完成后点 **Edit code**，进入代码编辑器

---

## 第三步：粘贴代理代码

1. 打开仓库里的 `worker/worker.js` 文件，复制全部内容
2. 粘贴到 Cloudflare 编辑器里（覆盖掉默认的 `hello world` 代码）
3. **不需要改任何东西**，密钥已经填好了
4. 点右上角 **Save and deploy**
5. 部署成功后，记下你的 Worker 地址，形如：
   ```
   https://zhiyu-proxy.你的用户名.workers.dev
   ```

---

## 第四步：启用 KV（频控存储，可选但推荐）

频控需要 KV 存储，防止恶意刷量。

1. 在 Cloudflare 控制台左侧，点 **Workers & Pages** → **KV**
2. 点 **Create a namespace**，名字填 `PROXY_KV`
3. 回到你的 Worker，点 **Settings** → **Variables** → **KV Namespace Bindings**
4. 添加绑定：
   - Variable name：`PROXY_KV`
   - KV namespace：选刚创建的 `PROXY_KV`
5. 点 **Save and deploy**

> 如果跳过这步，频控会失效，但代理仍能正常工作（不影响功能，只是没频控保护）。

---

## 第五步：把 Worker 地址填进前端

1. 打开 `/workspace/index.html`
2. 找到这行（约第 1303 行）：
   ```javascript
   const API_BASE = ''; // ← 部署 Cloudflare Worker 后填入地址
   ```
3. 把你的 Worker 地址填进去，例如：
   ```javascript
   const API_BASE = 'https://zhiyu-proxy.yourname.workers.dev';
   ```
4. 保存，提交，推送到 GitHub Pages

---

## 第六步：验证

1. 打开你的网站：https://yunmm123.github.io/Plant-Healing/
2. 打开任一植物详情，发一句话给植物
3. 应该能收到 AI 回复（如果没回复，按 F12 → Console 看报错）
4. 在设置里点「测试AI对话」按钮，应该显示 `✅ AI回复：...`
5. 在设置里填邮箱，点「发测试邮件」，应该能收到真实邮件

## 常见问题

### Q：部署后 AI 不回复？

A：检查三点：
1. `API_BASE` 是否填对（结尾不要带 `/`）
2. Worker 是否部署成功（访问 `https://你的地址/ai/chat` 应返回 404 JSON，不是空白）
3. 浏览器 F12 → Network 看 `/ai/chat` 请求的状态码和返回内容

### Q：邮件发不出去？

A：EmailJS 的 free 套餐每月只能发 200 封，且不能给同一邮箱频繁发。
检查 Worker 日志：Cloudflare 控制台 → 你的 Worker → **Logs** → 实时查看请求。

### Q：换密钥怎么办？

A：直接在 Cloudflare 编辑器里改 `worker.js` 顶部的常量，重新 Save and deploy 即可。
前端代码不用动。

### Q：Worker 会被刷爆吗？

A：免费额度每天 10 万次，加上内置频控（每 IP 每分钟 30 次），正常用不会超。
如果担心，可在 Cloudflare → Workers → 你的 Worker → **Triggers** 里开启 Rate Limiting。

### Q：能换其他代理服务吗？

A：可以。Vercel / Netlify / Deno Deploy 都行，逻辑一样：收前端请求，在服务端补 Key 转发。
只要把 `worker.js` 的逻辑改写成对应平台的函数格式即可。
